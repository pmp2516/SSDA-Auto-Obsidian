# Deep Work Protocol

> My personal implementation of Cal Newport's deep work philosophy. Refined over 3 months of experimentation.

## The Protocol

### Morning block (Mon–Fri, 08:00–12:00)
- Phone in bedroom until 12:00
- No Slack, email, or browser tabs except the one task I'm working on
- Water + coffee before sitting down; no refills until 12:00 (forces a break)
- Pomodoro optional — I use it for tasks with unclear scope; skip it for creative flow tasks

### Shutdown ritual (17:30–17:45)
1. Review open tabs — close all, copy anything important to notes
2. Check Slack for anything urgent (reply or schedule for tomorrow)
3. Review tomorrow's calendar
4. Write 3 tasks for tomorrow in daily note
5. Say aloud: "Shutdown complete."

The verbal ritual sounds silly but it works — it signals the brain that work mode is over. Without it, I kept thinking about work through the evening.

### Weekly planning (Sunday, 08:00–08:45)
- Review [[Weekly-Review-System]]
- Identify the week's "deep work anchor" — the one big output that matters most
- Time-block the deep work anchor into the morning blocks first, then fill other work around it

## What's Changed Since Implementing

| Metric | Before | After 3 months |
|---|---|---|
| Deep work hours/week | ~6h | ~18h |
| Evening phone use | ~2h/day | ~40min/day |
| Subjective focus quality | "Often distracted" | "Usually focused" |
| Writing output | ~800 words/week | ~2,400 words/week |

## Failures and Adjustments

- **Week 1–2**: Kept "just checking" Slack at 10:00. Fixed by moving laptop to dining table (away from usual desk = environmental cue for focused work).
- **Meeting mornings**: If a meeting is booked before 12:00, the whole morning is lost to anticipatory context-switching. Now I block mornings in calendar with "deep work — decline" and rebook essential meetings to 13:00–17:00.
- **Energy management**: Deep work after a poor sleep is counterproductive — shallow work is more resilient to tiredness. Added a 2-minute decision rule: if I slept < 6h, reschedule deep work task and do processing/email work instead.
