# perm-0051 — Mechanism lag: why conceptual books become useful after observation

**Core idea**: Books about mechanisms (how cognition works, how systems behave) have delayed practical usefulness because the reader needs to encounter the mechanism in the wild before the description becomes actionable. Books about frameworks (how to structure decisions or behaviour) are immediately applicable.

**Origin**: Fleeting note [[../00-Inbox/20260526-0913]] — observation that "Thinking, Fast and Slow" felt abstract while reading but became practically useful when I started recognising System 1 errors in real meetings.

**Proposed mechanism**: Reading a description of a cognitive bias gives you a label. But applying that label requires two additional steps: (1) noticing that an instance has occurred, and (2) recognising it as belonging to the labelled category. These pattern-matching skills only develop through practice — they cannot be simulated while reading.

This suggests an optimal learning strategy for mechanism books:
1. Read enough to understand the mechanism conceptually
2. Stop and observe (run meetings, have conversations, make decisions)
3. Return to the book to re-read with fresh context
4. Now the book reads differently — examples click that previously seemed abstract

**Contrast with framework books**: A decision matrix, a writing structure, a negotiation protocol — these are directly applicable because the framework *is* the action, not a description of how an action happens.

## Application

For [[../../../vault-1-PARA/Projects/Podcast-Series]] Ep 1: open with this distinction. "You will not be able to use this book immediately. Here's why that's actually correct and how to use it anyway."
